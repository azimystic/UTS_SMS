- Mobile menu: Full-screen overlay or slide-in drawer
- Navbar: Stack elements or hide some
- Search: Expandable or separate page

**Cards:**
- Stack from grid to single column
- Reduce padding (20px instead of 32px)
- Smaller font sizes

**Tables:**
- Transform to cards on mobile
- or Horizontal scroll with fixed first column
- Hide less important columns

**Forms:**
- Single column layout
- Full-width inputs
- Larger touch targets (48px minimum)
- Fixed action buttons at bottom

**Modals:**
- Full screen on mobile
- Slide up animation instead of scale

**Typography:**
- Reduce heading sizes (H1: 36px on mobile)
- Adjust line height for readability

### **10. ACCESSIBILITY**

**Requirements:**
- ARIA labels on all interactive elements
- Keyboard navigation support (Tab, Enter, Esc, Arrow keys)
- Focus visible indicators (outline:  2px solid #3B82F6)
- Alt text on all images
- Sufficient color contrast (WCAG AA minimum)
- Form labels properly associated with inputs
- Error messages announced by screen readers
- Skip to main content link
- Semantic HTML (header, nav, main, aside, footer)

**Focus Styles:**
```css
: focus-visible {
  outline: 2px solid #3B82F6;
  outline-offset: 2px;
}
```

### **11. PERFORMANCE OPTIMIZATIONS**

**CSS:**
- Use CSS variables for theme values
- Minimize use of expensive properties (box-shadow in moderation)
- Use transform for animations instead of position
- Implement critical CSS inline

**Images:**
- Optimize and compress all images
- Use appropriate formats (WebP with fallback)
- Lazy load images below the fold
- Use appropriate sizes (srcset)

**JavaScript:**
- Debounce search inputs
- Virtualize long lists/tables
- Lazy load modals and dropdowns
- Code splitting for routes

**Animations:**
- Use transform and opacity for smooth 60fps
- Respect prefers-reduced-motion
- Keep animations under 300ms
